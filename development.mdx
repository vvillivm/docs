<Info>
  **Prerequisite**:  Azure Admin / account owner privileges
</Info>

### **Requirements**

- Tenant ID
- Service Principle (Ginko_Connect)
- Secret Key
- Subscription ID
- Guest User (op@ginkocloud.com)

Both app and user require  "Reader" role to analyze your Azure resources.

### **Step 1**:  Tenant ID

1. Login to [<u>Azure Portal</u>](https://portal.azure.com)  →  **Microsoft Entra ID**
2. Click **Overview**
3. Copy **Tenant ID** (save for later)

![tenant_ID.png](/images/tenant_ID.png)

### **Step 2**:  Service Principle

 1. In **Microsoft Entra ID**
 2. Click **App registrations** → **New registration**

    ![new_app.png](/images/new_app.png)
 3. In **App registrations**, enter:

    **・ Name:** Ginko_Connection

    **・ Account type:** Single tenant (default)

    **・ Redirect URI:** (optional - leave blank)
 4. Click **Register**

    ![configure.png](/images/configure.png)
 5. Copy **Application ID** (save for later)
 6. Click **Client credentials**: Add a certificate or secret

    ![app_ID.png](/images/app_ID.png)
 7. Click **New client secret**
 8. In **Add a client secret**, enter:

    **・ Description:** GinkoKey

    **・ Expires:** 1 year (365 days)
 9. Click **Add**
10. Copy \*\*Secret Key ID \*\*(save for later)

    ![secret_key.png](/images/secret_key.png)

**Step 3**: Subscription ID (for Ginko API)

1. Navigate to **Subscriptions**
2. Click**a subscription**  to open the configuration page
3. Copy the Subscription ID (save for later) You will need the Subscription ID to initiate data collection, when using the Densify API.

We need the Subscription ID to initiate data collection when using the Ginko API.

```bash
mintlify dev --port 3333
```

If you attempt to run Mintlify on a port that's already in use, it will use the next available port:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify Versions

Please note that each CLI release is associated with a specific version of Mintlify. If your local website doesn't align with the production version, please update the CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```


```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>
    This may be due to an outdated version of node. Try the following:

    1. Remove the currently-installed version of mintlify: `npm remove -g mintlify`
    2. Upgrade to Node v19 or higher.
    3. Reinstall mintlify: `npm install -g mintlify`
  </Accordion>
  <Accordion title="Issue: Encountering an unknown error">
    Solution: Go to the root of your device and delete the ~/.mintlify folder. Afterwards, run `mintlify dev` again.
  </Accordion>
</AccordionGroup>
